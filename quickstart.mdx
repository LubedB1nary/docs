# Quick Start

This guide walks you through the entire flow from authentication to a submitted case. By the end, you'll have a working integration that creates a patient, submits a case for clinician review, and can track its progress.

## Before You Start

You need two things from the MD Integrations team:

- **Client ID** — identifies your partner account
- **Client Secret** — your private key for generating access tokens

If you don't have these yet, email [support@mdintegrations.com](mailto:support@mdintegrations.com) and they'll get you set up. They'll also configure your partner account with the offerings (medications, compounds, supplies) available to your patients.

## Step 1: Authenticate

The MDI API uses OAuth 2.0 Client Credentials. You exchange your Client ID and Secret for a bearer token that's valid for 24 hours.

<CodeGroup>

```bash cURL
curl -X POST https://api.mdintegrations.com/v1/partner/auth/token \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "client_id": "your-client-id",
    "client_secret": "your-client-secret"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.mdintegrations.com/v1/partner/auth/token",
    json={
        "client_id": "your-client-id",
        "client_secret": "your-client-secret"
    },
    headers={
        "Content-Type": "application/json",
        "Accept": "application/json"
    }
)
data = response.json()
token = data["access_token"]
```

```javascript JavaScript
const response = await fetch("https://api.mdintegrations.com/v1/partner/auth/token", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Accept": "application/json"
  },
  body: JSON.stringify({
    client_id: "your-client-id",
    client_secret: "your-client-secret"
  })
});
const { access_token } = await response.json();
```

```php PHP
<?php
$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, "https://api.mdintegrations.com/v1/partner/auth/token");
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Content-Type: application/json",
    "Accept: application/json"
]);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([
    "client_id" => "your-client-id",
    "client_secret" => "your-client-secret"
]));
$response = curl_exec($ch);
curl_close($ch);
$data = json_decode($response, true);
$token = $data["access_token"];
```

</CodeGroup>

You'll get back something like:

```json
{
  "token_type": "Bearer",
  "expires_in": 86400,
  "access_token": "eyJ0eXAiOiJKV1Q..."
}
```

Save that `access_token`. You'll include it as `Authorization: Bearer <token>` in every subsequent request. Tokens expire after 24 hours — when you get a `401` response, just request a new one.

## Step 2: Create a Patient

Before you can create a case, you need a patient. This call creates the patient record in both MDI and DoseSpot (the e-prescribing system) simultaneously.

<CodeGroup>

```bash cURL
curl -X POST https://api.mdintegrations.com/v1/partner/patients \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "first_name": "Jane",
    "last_name": "Smith",
    "email": "jane.smith@example.com",
    "date_of_birth": "1990-01-15",
    "gender": 2,
    "phone_number": "5551234567",
    "phone_type": 2,
    "address": {
      "address": "123 Main St",
      "city_name": "Austin",
      "state_name": "TX",
      "zip_code": "78701"
    }
  }'
```


</CodeGroup>

A few things to know about the patient payload:

- **`gender`** uses ISO 5218 format: `1` = Male, `2` = Female, `9` = Not specified, `0` = Unknown
- **`phone_type`** is `2` for cell or `4` for home
- **`email`** must be unique within your partner account — you can't create two patients with the same email
- **`address.state_name`** accepts either the full state name ("Texas") or the USPS abbreviation ("TX")

The response includes a `patient_id` (UUID) that you'll use for everything else. It also includes a `dosespot` object confirming the patient was synced to the e-prescribing system.

## Step 3: Create a Case

Now you can submit a case. A case represents a request for a clinician to review and prescribe treatment for the patient. You can include offerings (medications), questionnaire answers, files, and disease codes all in a single call.

<CodeGroup>

```bash cURL
curl -X POST https://api.mdintegrations.com/v1/partner/cases \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "patient_id": "PATIENT_ID_FROM_STEP_2",
    "metadata": "Order #12345",
    "case_offerings": [
      {
        "offering_id": "YOUR_OFFERING_ID",
        "product": {
          "quantity": 1,
          "days_supply": 30,
          "directions": "Take one tablet daily"
        }
      }
    ],
    "case_questions": [
      {
        "question": "Are you currently taking any medications?",
        "answer": "No",
        "type": "boolean",
        "important": true,
        "display_in_pdf": true
      }
    ]
  }'
```


Important things about the create case payload:

- **`metadata`** is your free-text field — store your internal order ID, Shopify order number, or whatever helps you correlate this case with your own systems. It gets included in webhook payloads.
- **`offering_id`** comes from `GET /v1/partner/offerings` — these are the medications, compounds, and supplies configured for your partner account.
- **`case_questions`** are the intake questionnaire answers. Each question has a `type` (`boolean`, `number`, or `string`) and you can flag critical ones with `important` and `is_critical`.
- **`hold_status`** (optional, defaults to `false`) — if set to `true`, the case won't automatically enter the MDI review flow. Useful if you need to add more information before it's ready.
- **`clinician_id`** (optional) — if you want a specific clinician. If omitted, MDI's algorithm assigns one automatically based on state licensing and availability.
- **`case_files`** (optional) — array of file UUIDs to attach. You need to upload files first via `POST /v1/partner/files`.

## Step 4: Check the Case Status

Once submitted, the case enters the MDI workflow. You can poll for status or (better) set up webhooks.

<CodeGroup>

```bash cURL
curl -X GET https://api.mdintegrations.com/v1/partner/cases/CASE_ID \
  -H "Accept: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
import requests

case_details = requests.get(
    f"https://api.mdintegrations.com/v1/partner/cases/{case_id}",
    headers={
        "Authorization": f"Bearer {token}",
        "Accept": "application/json"
    }
)
status = case_details.json()["case_status"]["name"]
print(f"Case status: {status}")
```

```javascript JavaScript
const details = await fetch(`https://api.mdintegrations.com/v1/partner/cases/${case_id}`, {
  headers: {
    "Authorization": `Bearer ${access_token}`,
    "Accept": "application/json"
  }
});
const { case_status } = await details.json();
console.log(`Case status: ${case_status.name}`);
```

```php PHP
<?php
$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, "https://api.mdintegrations.com/v1/partner/cases/$case_id");
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer $token",
    "Accept: application/json"
]);
$response = curl_exec($ch);
curl_close($ch);
$case = json_decode($response, true);
echo "Case status: " . $case["case_status"]["name"];
```

</CodeGroup>

The response includes everything: patient details, assigned clinician, offerings with their statuses, case status, and timestamps.

## That's It!

You've just completed the core integration flow. Here's what typically happens next:

1. MDI assigns a clinician (usually within minutes)
2. The clinician reviews the case and either approves it, requests more info, or denies it
3. If approved, prescriptions are submitted to the pharmacy via DoseSpot
4. The pharmacy dispenses and ships the medication
5. You receive webhook notifications at every step
