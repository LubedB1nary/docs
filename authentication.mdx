# Authentication

The MDI API uses **OAuth 2.0 Client Credentials** — the same pattern used by Stripe, Twilio, and most modern APIs. You exchange a client ID and secret for a time-limited bearer token, then include that token in every request.

## Getting Your Credentials

When you sign up as an MDI partner, the team provides you with two credentials:

- **Client ID** — a public identifier for your partner account
- **Client Secret** — a private key used to generate tokens

<Warning>
  Your Client Secret is sensitive. Never expose it in client-side code, public repositories, or browser-accessible JavaScript. Treat it like a database password.
</Warning>

## Generating an Access Token

Exchange your credentials for a bearer token:

<CodeGroup>

```bash cURL
curl -X POST https://api.mdintegrations.com/v1/partner/auth/token \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "client_id": "your-client-id",
    "client_secret": "your-client-secret"
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.mdintegrations.com/v1/partner/auth/token",
    json={
        "client_id": "your-client-id",
        "client_secret": "your-client-secret"
    },
    headers={"Content-Type": "application/json", "Accept": "application/json"}
)
data = response.json()
token = data["access_token"]
# Token expires in data["expires_in"] seconds (typically 86400 = 24 hours)
```

```javascript JavaScript
const response = await fetch("https://api.mdintegrations.com/v1/partner/auth/token", {
  method: "POST",
  headers: { "Content-Type": "application/json", "Accept": "application/json" },
  body: JSON.stringify({
    client_id: "your-client-id",
    client_secret: "your-client-secret"
  })
});
const { access_token, expires_in } = await response.json();
// Store token and schedule refresh before expires_in seconds
```

```php PHP
<?php
$ch = curl_init("https://api.mdintegrations.com/v1/partner/auth/token");
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Content-Type: application/json",
    "Accept: application/json"
]);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([
    "client_id" => "your-client-id",
    "client_secret" => "your-client-secret"
]));
$data = json_decode(curl_exec($ch), true);
curl_close($ch);
$token = $data["access_token"];
```

</CodeGroup>

The response:

```json
{
  "token_type": "Bearer",
  "expires_in": 86400,
  "access_token": "eyJ0eXAiOiJKV1Q..."
}
```

## Using the Token

Include the token in the `Authorization` header of every request:

```http
GET /v1/partner HTTP/1.1
Host: api.mdintegrations.com
Content-Type: application/json
Accept: application/json
Authorization: Bearer eyJ0eXAiOiJKV1Q...
```

## Handling Token Expiration

Tokens expire after `expires_in` seconds (typically 24 hours). When a token expires, the API returns `401 Unauthorized`. Your integration should handle this gracefully:

1. Catch the `401` response
2. Request a new token using the same client ID and secret
3. Retry the failed request with the new token

<Tip>
  A common pattern is to cache the token and proactively refresh it a few minutes before expiration, rather than waiting for a 401. This avoids any interruption in service.
</Tip>

## Environments

MDI uses the same base URL for both production and sandbox. The environment is determined by your credentials — sandbox credentials create sandbox data that doesn't trigger real prescriptions or charges.

| Environment | Base URL | Behavior |
|------------|----------|----------|
| **Sandbox** | `https://api.mdintegrations.com/v1` | Test data only. No real prescriptions or charges. |
| **Production** | `https://api.mdintegrations.com/v1` | Live data. Real prescriptions sent to pharmacies. |

Your MDI team will provide separate credentials for each environment.
